OVERVIEW
--------
AnbuDevs is a comprehensive developer skills platform combining coding practice
(3000+ problems), aptitude quizzes, communication training, tutorials, career
roadmaps, and gamification — built with React + TypeScript + Vite + Tailwind CSS.


================================================================================
1. LOADING SCREEN — CRITICAL FIX
================================================================================

ISSUE:
  The 6.5-second forced splash screen is the biggest UX problem. Users cannot
  skip it, and it loads on every visit. This will cause high bounce rates.

FIX:
  - Reduce to 1.5-2 seconds max
  - Add a "Skip" button
  - Only show the full animation on first visit (use localStorage)
  - Show actual loading progress instead of a fixed timer


================================================================================
2. SEO & PERFORMANCE
================================================================================

ISSUES:
  - No SSR/SSG: Being a pure SPA, search engines will struggle to index content
  - Monaco editor is a large bundle loaded upfront
  - No meta tags for social sharing
  - No sitemap.xml or robots.txt

FIXES:
  - Consider migrating to Next.js or adding pre-rendering
  - Lazy load images and heavy components (especially Monaco editor)
  - Add Open Graph and Twitter card meta tags for social sharing
  - Add a sitemap.xml and robots.txt


================================================================================
3. AUTHENTICATION & USER ACCOUNTS
================================================================================

ISSUE:
  Currently all data is stored in localStorage only — users lose everything
  if they clear their browser or switch devices.

ADD:
  - User authentication (Firebase Auth, Supabase, or Clerk)
  - Cloud sync for progress, submissions, and solved problems
  - User profiles with avatars


================================================================================
4. BACKEND/DATABASE INTEGRATION
================================================================================

ISSUES:
  - The leaderboard uses mock data — not connected to a real backend
  - Submission history only persists in localStorage
  - Code execution uses simulated results — not using actual Judge0 API

FIXES:
  - Connect leaderboard to a real backend/database
  - Persist submission history server-side
  - Integrate the real Judge0 API or use Piston API for code execution
